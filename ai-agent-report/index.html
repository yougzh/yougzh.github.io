1. AI Agent 架构方案

2025-2026 年，Agent 架构不再是"选一种就行"，而是多种模式融合。以下是五类核心架构范式的深度分析。

1.1 ReAct 循环架构

Thought → Action → Observation 交替进行，形成闭环推理。LangGraph 将此抽象为有向状态图（DAG），支持循环、条件分支和并行。

✓ 可解释性强 ✓ 错误可恢复 ✕ Token 消耗大 ✕ 长任务退化
langchain-ai/langgraph⭐ 36,702
run-llama/llama_index⭐ 50,704
SWE-agent/SWE-agent⭐ 19,724

论文: Yao et al., "ReAct: Synergizing Reasoning and Acting in Language Models", ICLR 2023

1.2 Plan-and-Execute

先规划后执行：Planner 生成完整计划 → Executor 逐步执行 → 可选 Replan。适合长篇写作、深度研究等需要全局规划的场景。

✓ 全局最优 ✓ 结构化输出 ✕ 灵活性差 ✕ 初始偏差传播
stanford-oval/storm⭐ 29,887
assafelovic/gpt-researcher⭐ 28,133
Codium-ai/AlphaCodium⭐ 3,946

STORM: Shao et al., NAACL 2024 · AlphaCodium: Codeforces 基准

1.3 Multi-Agent 协作

多个专业化 Agent 通过通信协作。三种模式：Swarm（涌现）、Hierarchical（层级 GroupChat）、Collaborative（SOP 角色扮演）。

✓ 专业化分工 ✓ 高鲁棒性 ✕ 通信开销大 ✕ 协调困难
microsoft/autogen⭐ 59,552
crewAIInc/crewAI⭐ 55,076
geekan/MetaGPT⭐ 69,243

MetaGPT: Hong et al., ICLR 2024 · AutoGen: Wu et al., 2023

1.4 Tool-use / Function Calling

LLM 识别何时调用外部工具，以 JSON Schema 输出函数调用。OpenAI / Anthropic / Google 已形成事实标准，MCP 协议正在统一工具通信。

✓ 扩展能力边界 ✓ API 标准化 ✕ 幻觉风险 ✕ 安全风险
langchain-ai/langchain⭐ ~141K
openinterpreter/open-interpreter⭐ 64,296
BerriAI/litellm⭐ 52,859

论文: Schick et al., "Toolformer", NeurIPS 2023 · Gorilla, ICML 2024

1.5 认知架构

模拟人脑认知过程：工作记忆 + 长期记忆 + 目标系统 + 注意力。Soar (MIT)、ACT-R (CMU) 为经典，现代变体如 Letta (虚拟 OS 记忆)、Generative Agents。

✓ 理论严谨 ✓ 记忆系统完善 ✕ 实现复杂 ✕ 与 LLM 范式存在张力
mem0ai/mem0⭐ ~25K
letta-ai/letta (原 MemGPT)⭐ ~14K
joonspk-research/generative_agents

Park et al., "Generative Agents", UIST 2023 · Packer et al., "MemGPT", NeurIPS 2024

2. 记忆与上下文管理

记忆是 Agent 从"一次性工具"进化为"持久化助手"的关键。以下覆盖短期、长期、混合、优化、外部工具五大领域。

2.1 短期/工作记忆

滑动窗口 (Sliding Window)

仅保留最近 N 轮对话 / K 个 token，超出直接丢弃。实现极简，零额外开销。

✓ 零延迟 ✓ 零成本 ✕ 丢失早期信息 ✕ 无法跨多轮连贯

LangChain ConversationBufferWindowMemory · LlamaIndex ChatMemoryBuffer

摘要记忆 (Summarization)

历史超出窗口时，调用 LLM 生成摘要替代原始内容。保留语义精髓，但可能丢失细节。

✓ 语义保留 ✓ Token 节省 ✕ 额外 LLM 调用 ✕ 细节可能丢失

LangChain ConversationSummaryMemory · LangGraph MemorySaver

结构化工作记忆 (Scratchpad)

给 Agent 提供"草稿纸"区域，记录推理步骤、子目标、中间结果。AutoGPT / BabyAGI 经典模式。

✓ 推理透明 ✓ 子目标追踪 ✕ 占用上下文 ✕ 需人工设计结构
Significant-Gravitas/AutoGPT⭐ ~172K
yoheinakajima/babyagi⭐ ~20K

2.2 长期记忆

向量存储 / 语义记忆

文本 → Embedding → 向量数据库，通过语义相似度检索。支持海量条目，模糊匹配。

✓ 语义检索 ✓ 海量可扩展 ✕ Embedding 成本 ✕ Chunking 需调优
chroma-core/chroma⭐ 28,721
qdrant/qdrant (Rust, 混合搜索)
milvus-io/milvus (十亿级)
facebookresearch/faiss (极限性能)

情景记忆 (Episodic)

记录 Agent 的具体"经历"（何时、何事、何果），用于"从经验中学习"。Voyager 在 Minecraft 中实现终身学习。

✓ 从经验学习 ✓ 自动 Few-shot ✕ 经验提取 pipeline 复杂 ✕ 存储线性增长
MineDojo/Voyager (Minecraft 终身学习)
Generative Agents (Stanford 小镇)

知识图谱记忆 (KG)

实体-关系三元组存储，支持精确符号推理和多跳查询。Cognee 实现图谱+向量融合。

✓ 精确关系推理 ✓ 多跳查询 ✕ 三元组提取成本高 ✕ 构建维护复杂
topoteretes/cognee (图谱+向量融合)
LangChain GraphCypherQAChain

2.3 混合记忆 — MemGPT / Letta (OS 风格)

Letta (原 MemGPT)

借鉴操作系统虚拟内存管理：LLM 上下文 = RAM，外部存储 = Disk。Agent 自主决定何时换出/加载记忆。2024年商业化后核心框架仍开源。

✓ 突破窗口限制 ✓ 跨会话持久化 ✕ 额外 LLM 调用 ✕ 召回可能不完整
letta-ai/letta⭐ ~14K

Mem0 — Agent 记忆层

专为 Agent 设计的智能记忆层。自动提取偏好/事实，多级记忆（用户/会话/Agent），支持去重、更新、时间衰减、图记忆。

✓ 自动记忆管理 ✓ 少量代码集成 ✕ 托管版有成本 ✕ 封装度高
mem0ai/mem0⭐ ~25K

2.4 上下文窗口优化

Token 压缩

Microsoft LLMLingua 用小模型有损压缩提示词，LongLLMLingua 支持超长上下文。最高压缩至 1/20。

microsoft/LLMLingua
liuy967/selective_context

上下文缓存

Anthropic Prompt Caching 省 90% 成本、Google Context Caching、vLLM APC 复用 KV Cache。无损优化，显著降低首 Token 延迟。

vllm-project/vllm (APC)
sgl-project/sglang (RadixAttention)

结构化提示

DSPy (Stanford) 自动优化提示词结构，Guidance (Microsoft) 提供约束生成。XML/JSON 语义分块 + 位置感知。

stanfordnlp/dspy
guidance-ai/guidance
jxnl/instructor (Pydantic 约束)

3. RAG 实现方案

检索增强生成已经从"一个 pipeline"进化为"一套方法论"。以下是五类 RAG 范式的深度对比。

3.1 Naive RAG (基础)

最简单的 RAG 流水线：文档分块 → Embedding → 向量检索 → Top-K 注入 Prompt → LLM 生成。LangChain / LlamaIndex / Chroma 提供开箱即用的实现。

✓ 实现简单 ✓ 快速上手 ✕ 低召回率 ✕ 块粒度粗 ✕ 无重排序
langchain-ai/langchain⭐ 141,194
run-llama/llama_index⭐ 50,704
chroma-core/chroma⭐ 28,721

3.2 Advanced RAG (进阶)

小块检索 + 大块生成 (Small-to-Big)、句子窗口检索、自动合并 (Auto-Merging)、重排序 (Rerank)。检索精度大幅提升，但复杂度增加。

✓ 召回精度高 ✓ 上下文连贯 ✕ 实现复杂度 ↑ ✕ 多步检索延迟 ↑
infiniflow/ragflow⭐ 84,510
LlamaIndex (SentenceWindowNodeParser)
cohere-ai/cohere-toolkit⭐ 3,176

3.3 Agentic RAG (智能体)

RAG + Agent 融合：多步检索（迭代查询）、自我反思 (Self-RAG)、自适应路由（根据问题类型选检索策略）、工具调用。

✓ 复杂查询处理 ✓ 自我纠错 ✕ Token 消耗极高 ✕ 延迟大
langchain-ai/langgraph⭐ 36,702
crewAIInc/crewAI⭐ 55,076
RAGFlow (内置 Agentic 模式)

3.4 Graph RAG (知识图谱)

知识图谱 + RAG：先构建实体-关系图，社区检测提取摘要，图遍历检索相关信息。处理全局性/总结性问题远超传统 RAG。

✓ 全局总结能力 ✓ 关系推理 ✕ 构建成本极高 ✕ 实时更新困难
microsoft/graphrag⭐ 34,237
HKUDS/LightRAG (港大, 轻量)
LlamaIndex KnowledgeGraphRAG

3.5 Hybrid RAG (混合)

稠密向量检索 (Dense) + 稀疏检索/BM25 (Sparse) + 重排序 (RRF/Cross-Encoder)。结合语义理解和关键词匹配，业界最稳健方案。

✓ 兼顾语义+关键词 ✓ 最稳健 ✕ 多索引维护 ✕ 延迟较高
weaviate/weaviate⭐ 16,533
qdrant/qdrant (原生混合搜索)
milvus-io/milvus (多向量字段)

5. 2025-2026 关键趋势

1. 架构融合

现代 Agent 框架不再纯用单一架构。LangGraph 融合了 ReAct + Plan-Execute；AutoGen 融合 Multi-Agent + Tool-use。未来是"混合架构"的时代。

2. MCP 协议标准化

Anthropic 推动的 Model Context Protocol 正在成为 Agent-工具通信的事实标准。记忆工具、向量库、API 网关纷纷接入 MCP，工具生态互通时代来临。

3. OS 化记忆管理

MemGPT/Letta 的"LLM 自治记忆"范式成为主流方向。Agent 不再被动接受记忆管理，而是主动决定存储/检索/遗忘。Agent-Native 记忆工具 (Mem0/Cognee) 将取代通用向量库。

4. Agentic Coding

Devin / Cursor / Copilot 将 Agent 架构深度融入 IDE，ReAct 循环成为默认交互模式。MCP 协议让 IDE Agent 可以安全调用文件系统、数据库、API。

5. Computer Use / GUI Agent

Claude Computer Use、OpenAI Operator 将 Agent 从 API 扩展到 GUI 操作。多模态 + 像素级理解 + 动作空间定义是新的技术高地。

6. Prompt Caching 普及

Anthropic / Google / vLLM 的 KV Cache 复用方案将大幅降低 Agent 运行成本（最高省 90%）。无损优化 + 显著降低首 Token 延迟。

7. 多级记忆融合

短期 + 长期 + 情景 + 语义四层记忆架构成为标配。向量存储负责"知道什么"，KG 负责"知道关系"，情景记忆负责"经历过什么"。

8. RAG 进入 Agentic 时代

RAG 从静态检索升级为动态 Agentic 检索——多步推理 + 自适应路由 + 自我反思 (Self-RAG) + 工具编排。Graph RAG 解决全局性问题，Hybrid RAG 成为生产基线。
